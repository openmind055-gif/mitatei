# 三田丁LP 多言語対応 引継ぎ資料

更新日: 2026-07-04

## 1. プロジェクト概要

- プロジェクト: 石垣島のしゃぶしゃぶ専門店「三田丁」のランディングページ
- フレームワーク: Next.js 16.2.9（App Router）
- React: 19.2.4
- スタイリング: Tailwind CSS 4
- 公開中の参考URL: https://mitatei.vercel.app/
- ローカルプロジェクト: `C:\Users\user\Desktop\workspace\projects\mitatei-lp`

## 2. 今回の依頼内容

既存デザインを維持したまま、LPを日本語・英語の2言語に対応する。

要件:

- 初期表示は日本語
- JP/ENボタンで表示言語を切り替える
- 英語は飲食店LPとして自然な表現にする
- まずは `app/page.tsx` 内で完結させる
- 韓国語・中国語を後から追加しやすい構造にする

## 3. 実装済みの内容

変更対象は `app/page.tsx` のみ。

### 言語辞書

ファイル冒頭に `languages` オブジェクトを追加した。

```ts
const languages = {
  ja: {
    label: "JP",
    htmlLang: "ja",
    reservationUrl: "...",
    messages: { /* 日本語文言 */ },
  },
  en: {
    label: "EN",
    htmlLang: "en",
    reservationUrl: "...",
    messages: { /* 英語文言 */ },
  },
} as const;

type Locale = keyof typeof languages;
```

辞書には以下の文言を収録している。

- ナビゲーション
- ヒーロー
- コンセプト
- 食材紹介
- 店内空間
- 店舗情報・アクセス
- 予約セクション
- フッター
- 画像の代替テキスト
- アクセシビリティ用ラベル
- 言語別のTableCheck予約URL

### 言語切り替え

`Home` コンポーネント内に次のstateを追加した。

```ts
const [locale, setLocale] = useState<Locale>("ja");
const language = languages[locale];
const t = language.messages;
```

初期値が `"ja"` のため、初期表示は日本語。

ヘッダーの旧表示 `JP/EN/KR/CN` を、クリック可能なJP/ENボタンへ変更した。ボタン一覧は `Object.keys(languages)` から生成しているため、言語辞書を追加するとボタンも自動的に増える。

選択中の言語には `aria-pressed` を設定し、色でも選択状態が分かるようにしている。

### ページ言語属性

言語切り替え時に、ブラウザ上の `<html lang>` も更新する。

```ts
useEffect(() => {
  document.documentElement.lang = language.htmlLang;
}, [language.htmlLang]);
```

### 英語コピー

日本語の直訳ではなく、海外旅行者向け飲食店LPとして自然に読めるコピーへ調整した。

例:

- `三田丁の体験` → `The Mitatei Experience`
- `厳選された至高の食材` → `Exceptional Island Ingredients`
- `五感で愉しむ空間` → `A Feast for Every Sense`
- `空席を確認・予約する` → `Check Availability & Book`

Ishigaki Beef、Okinawan Agu Pork、Sea Grapesなど、旅行者が内容を理解しやすい名称を使用している。

### 予約ページ

表示言語に合わせてTableCheckのリンクも切り替わる。

- 日本語: `/ja/mitatei-ishigaki/...`
- 英語: `/en/mitatei-ishigaki/...`

電話リンク `tel:0980826351` は共通。

### デザイン維持

既存のセクション構造、画像、色、余白、グリッド、アニメーション用クラスは維持している。基本的には固定文字列を辞書参照へ置換した。

モバイルでも言語ボタンを使えるよう、ヘッダーのみ次のレスポンシブ調整を行った。

- ロゴ幅: モバイル100px、`sm`以上140px
- ヘッダー内の間隔をモバイル向けに縮小
- 予約ボタンの横余白と文字サイズをモバイル向けに縮小
- JP/ENボタンをモバイルでも表示

## 6. 韓国語・中国語を追加する方法

`languages` に日本語・英語と同じ構造のエントリを追加する。

```ts
const languages = {
  ja: { /* ... */ },
  en: { /* ... */ },
  ko: {
    label: "KR",
    htmlLang: "ko",
    reservationUrl: "韓国語版の予約URL",
    messages: {
      // ja/enと同じキーをすべて用意
    },
  },
  zh: {
    label: "CN",
    htmlLang: "zh-CN",
    reservationUrl: "中国語版の予約URL",
    messages: {
      // ja/enと同じキーをすべて用意
    },
  },
} as const;
```

ボタン側の修正は不要。`Locale` 型も自動的に `"ja" | "en" | "ko" | "zh"` へ拡張される。

追加時は、すべての言語で `messages` のキー構造を一致させること。

## 7. 未対応・今後検討したい点

### `app/layout.tsx` のメタデータ

現在のメタデータはCreate Next Appの初期値のまま。

```ts
export const metadata: Metadata = {
  title: "Create Next App",
  description: "Generated by create next app",
};
```

SEOのため、少なくとも三田丁向けのタイトルと説明文へ変更する必要がある。

また、サーバーから返る初期HTMLの `<html lang>` は現在 `en`。画面表示後に `page.tsx` の `useEffect` で `ja` へ変わるが、初期表示とSEOを厳密に合わせるなら `app/layout.tsx` の初期値を `ja` に変更する。

### 言語選択の保存

現在はページを再読み込みすると日本語へ戻る。選択言語を維持したい場合は、次のいずれかを検討する。

- `localStorage`
- Cookie
- URLパス方式（例: `/ja`, `/en`）
- App Routerの `[locale]` 動的セグメント

本格的なSEO・言語別URLが必要になった段階では、`app/[locale]/page.tsx` と辞書ファイル分割への移行が望ましい。

### Googleマップ

埋め込みURL自体は日本語設定のまま。周辺の見出しや案内文は英語へ切り替わる。マップUIも言語別に切り替えたい場合は、言語別の埋め込みURLを辞書へ追加する。

### 画面確認

Lintと本番ビルドは完了している。デプロイ後は、実端末またはブラウザで次を最終確認すること。

- JP → EN → JPの連続切り替え
- 375px前後のモバイルヘッダー
- 英語表示時の長い見出し・予約ボタン
- TableCheckが選択言語のページで開くこと
- Googleマップ表示
